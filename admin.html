📌 Tiga Endpoint yang Tersedia

1. Data Penjualan

Memuat...

Parameter: p_api_key (wajib), p_date_from, p_date_to, p_branch_id (semua opsional kecuali p_api_key)

Mengembalikan semua transaksi completed. Void tidak disertakan.

2. Data Kas Keluar

Memuat...

Parameter: sama seperti endpoint penjualan.

Mengembalikan semua kas keluar yang sudah tercatat (non-void).

3. Ringkasan Gabungan

Memuat...

Parameter: sama seperti endpoint penjualan.

Mengembalikan total, selisih, per-cabang, dan per-tanggal sekaligus.

💡 Catatan Penting untuk AI Builder & Developer

  • Semua waktu menggunakan timezone WITA — Asia/Makassar (UTC+8)
  • Data penjualan hanya status "completed" — transaksi void tidak disertakan
  • Data kas keluar hanya yang belum di-void (is_void = false)
  • Jika tidak ada data, field data berisi array kosong []
  • Parameter tanggal bersifat opsional — jika kosong, semua data dikembalikan
  • Rentang tanggal maksimal 365 hari per request (untuk per_tanggal di summary)
  • Gunakan selisih di endpoint ringkasan untuk kalkulasi profit kasar langsung
  • API key bersifat rahasia — jangan ekspose di frontend publik
